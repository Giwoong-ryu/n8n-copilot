프로젝트 개요

AI 기반 워크플로우 자동화를 통해 개발자의 생산성을 극대화합니다

문제 정의

n8n 워크플로우 작성 시 반복적인 설정과 복잡한 노드 구성으로 인해 개발 시간이 과도하게 소요되고, 휴먼 에러가 빈번하게 발생합니다.

솔루션

Chrome Extension과 AI를 결합하여 실시간 페이지 분석, 자동 입력, 에러 감지를 제공하며, Vue.js 리액티브 시스템과 완벽하게 통합됩니다.

주요 목표

반복 작업 자동화를 통한 개발 시간 단축, 안정적인 DOM 조작 패턴 구축, 그리고 재사용 가능한 워크플로우 템플릿 라이브러리 제공을 목표로 합니다.

개발 과정

체계적인 접근과 반복적 개선

Phase 1

요구사항 분석 및 설계

Phase 2

코드 리뷰 및 검증

Phase 3

Production-Ready 구현

Phase 4

문서화 및 배포

code-changelog 자동 생성, HTML 뷰어 제작, GitHub Pages 배포로 포트폴리오화

핵심 기능

실시간 페이지 분석부터 자동화까지

실시간 페이지 분석

n8n 페이지의 DOM 구조, CSS 클래스, data-test-id 속성을 실시간으로 분석하여 올바른 선택자를 자동으로 찾습니다.

Vue 리액티브 통합

6단계 프로세스(focus, select, setValue, input event, change event, blur)를 통해 Vue.js 리액티브 시스템을 완벽하게 트리거합니다.

SafeSelector 패턴

여러 우선순위의 선택자를 시도하여 n8n 버전 변경에도 안정적으로 동작하는 Fallback 시스템을 제공합니다.

자동 재시도 메커니즘

네트워크 지연이나 DOM 준비 지연 시 exponential backoff 방식으로 자동 재시도하여 안정성을 보장합니다.

기술적 도전과 해결

실전에서 마주한 문제들과 창의적 솔루션

Vue.js 리액티브 미작동

문제: element.value = x 방식이 Vue Proxy를 트리거하지 못함
해결: Native setter + 이벤트 체인 + Vue 인스턴스 직접 업데이트

MutationObserver 메모리 누수

문제: 페이지 언로드 후에도 Observer가 작동하여 40MB/시간 누수
해결: beforeunload 이벤트에서 disconnect() 호출로 완전 정리

선택자 취약성

문제: n8n 버전 업데이트 시 하드코딩된 선택자 실패
해결: SafeSelector 패턴으로 우선순위 기반 Fallback 구현

보안 취약점

문제: Message 이벤트에 origin 검증 없어 외부 주입 가능
해결: event.source === window 체크로 Same-origin Policy 적용
