**이메일 종류:**
- [ ] 업무 요청 이메일
- [ ] 회의 일정 조율
- [ ] 프로젝트 진행 상황 공유

- 정중하지만 간결하게
- 수동적이지 않고 능동적으로

📖 초보자를 위한 용어 사전

터미널 (Terminal)
= 명령 프롬프트 = 명령줄
컴퓨터에게 명령어로 지시하는 프로그램
Windows: cmd 또는 PowerShell / Mac: 터미널
커밋 (Commit)
= 저장 = 버전 기록
코드 변경 사항을 기록으로 남기는 것
게임 저장처럼, 이전 시점으로 돌아갈 수 있음
푸시 (Push)
= 업로드 = 동기화
내 컴퓨터의 변경사항을 서버에 올리는 것
구글 드라이브에 파일 올리는 것과 비슷
풀 (Pull)
= 다운로드 = 최신화
서버의 최신 코드를 내 컴퓨터로 가져오는 것
클라우드에서 파일 받는 것과 비슷
브랜치 (Branch)
= 가지 = 작업 공간
원본을 건드리지 않고 새 기능 개발하는 공간
문서 사본 만들어서 편집하는 것과 비슷
리포지토리 (Repository)
= 저장소 = 프로젝트 폴더
코드와 변경 기록이 모두 저장된 공간
프로젝트 전체를 담는 구글 드라이브 폴더
CLI (Command Line Interface)
= 명령줄 인터페이스
마우스 대신 키보드로 명령어 입력하는 방식
터미널에서 명령어 치는 것
토큰 (Token)
= AI가 읽는 단어 수 = 사용량
Claude가 처리하는 텍스트의 양
한글 1글자 ≈ 3-4토큰, 영어 1단어 ≈ 1토큰

→ MCP 서버 연결
예시: 프로젝트 파일 자동 읽기, Git 저장소 조회, 데이터베이스 쿼리
🚀 복잡한 프로젝트를 한 번에 처리할 때
→ Agents 활용
예시: 전체 앱 개발 → fullstack-developer Agent / 코드 리뷰 → code-reviewer Agent
🎨 특정 언어/프레임워크 전문가가 필요할 때
→ Plugins 설치
예시: React 19 개발 → frontend-excellence 플러그인 / Git 자동화 → commit-commands 플러그인

❌ 실수 1: Skills 전부 로드

✅ 해결: 필요한 스킬만 5-10개 선택 로드 (예: n8n 개발자는 n8n 패키지 6개만)

✅ 해결: 큰 작업 끝나면 새 채팅 시작 or /index로 컨텍스트 리셋

❌ 실수 3: MCP 없이 반복 데이터 조회

✅ 해결: MCP 파일시스템 서버 설정 → 필요할 때만 자동 조회 (5K 토큰/일)

❌ 실수 4: 같은 질문 반복

문제: "커밋 메시지 작성해줘"를 하루에 20번 → 40K 토큰 낭비
✅ 해결: /git Commands 만들기 → 10K 토큰으로 감소 (75% 절약)

💡 시나리오 1: n8n 워크플로우 5개 개발

❌ Skills 없이 개발
• 매번 n8n 개념, 노드, 패턴 설명 요청
• 워크플로우당 10K 토큰 × 5개 = 50K 토큰
• 각 워크플로우마다 반복 설명 필요
✅ Skills (n8n 패키지 6개) 사용
• Skills 한 번 로드: 20K 토큰
• 워크플로우당 1K 토큰 × 5개 = 5K 토큰
• 총 25K 토큰 (절약: 25K 토큰 / 50%)

💡 시나리오 2: Git 커밋 & PR 생성 (10회 반복)

❌ Commands 없이 수동
• 매번 "커밋 메시지 작성해줘", "PR 생성해줘" 요청
• 회당 2K 토큰 × 10회 = 20K 토큰
✅ Commands (/git) 사용
• Commands 정의: 0 토큰
• 회당 0.5K 토큰 × 10회 = 5K 토큰
• 총 5K 토큰 (절약: 15K 토큰 / 75%)

💡 시나리오 3: 복잡한 전체 기능 개발 (Agent 활용)

❌ 일반 대화로 단계별 개발
• 1단계: 설계 요청 (10K)
• 2단계: 백엔드 구현 (20K)
• 3단계: 프론트 구현 (20K)
• 4단계: 테스트 작성 (15K)
• 5단계: 문서화 (10K)
• 총 75K 토큰 + 컨텍스트 누적
✅ Agent (fullstack-developer) 사용
• Agent 정의: 0 토큰
• 한 번에 전체 작업 요청: 40K 토큰
• 총 40K 토큰 (절약: 35K 토큰 / 47%)
• 보너스: 컨텍스트 관리 자동화

💡 시나리오 4: 실시간 파일 조회 (10개 파일, 매일 최신 데이터)

❌ Skills로 정적 로드
• 파일 내용 Skills로 저장: 30K 토큰
• 매일 새로운 세션에 로드: 30K × 7일 = 210K 토큰/주
• 문제: 데이터 변경 시 Skills 업데이트 필요
✅ MCP 서버 사용
• MCP 서버 설정: 0 토큰
• 필요할 때만 조회: 5K × 7일 = 35K 토큰/주
• 절약: 175K 토큰/주 / 83%
• 보너스: 항상 최신 데이터 접근

💡 토큰 절약 전략

1. Skills 선택적 로드: 필요한 스킬만 패키지로 묶어 사용 (전체 대신 5-10개) → 50% 절약

3. Agents 효율 사용: 복잡한 멀티스텝 작업은 Agent로 한 번에 → 40-50% 절약
4. MCP 실시간 데이터: 자주 변경되는 데이터는 MCP로 → 80% 절약
5. 요금제 선택: 프로젝트 규모에 맞는 요금제 (Free→Pro→Max)
