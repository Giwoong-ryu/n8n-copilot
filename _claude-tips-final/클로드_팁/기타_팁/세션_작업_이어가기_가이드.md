# Claude Code 세션 작업 이어가기 가이드

## 📝 개요

Claude Code Desktop에서 새 세션을 시작할 때 이전 작업을 효과적으로 이어가는 방법

---

## 🎯 핵심 원칙

1. **구체적인 파일 경로** 제공
2. **마지막 작업 내용** 명시
3. **다음 작업 목표** 언급
4. **관련 문서** 참조

---

## 🔄 요청 방법 (난이도별)

### Level 1: 간단한 요청 (초보자)

```
지난 번에 하던 작업 이어서 해줘
```

**장점**: 간단함
**단점**: Claude가 파일 검색에 시간 소요

---

### Level 2: 키워드 제공 (중급자)

```
[프로젝트명] 작업 이어가려고 해.
마지막으로 [완료 작업]까지 했어.
```

**예시**:
```
n8n-skillset 패키지 작업 이어가려고 해.
마지막으로 Package 1 완성했어.
```

---

### Level 3: 파일 경로 제공 (고급자 ⭐ 추천)

```
[파일 경로] 보고 [섹션명] 작업 이어가줘
```

**예시**:
```
docs/plugin-design-proposal.md 보고
"다음 단계" 섹션의 2번 작업 시작해줘
```

---

### Level 4: 세션 백업 활용 (전문가)

```
claude-sessions-backup/[세션파일].md 읽고
마지막 TODO 이어가줘
```

**예시**:
```
claude-sessions-backup/claude-conversation-2025-11-06-5b55f95c.md
읽고 마지막 작업 이어가줘
```

## 🎯 상황별 최적 방법

### 상황 1: 명확한 문서가 있을 때

**최선**: 문서 경로 제공
```
docs/plugin-design-proposal.md의
"다음 단계" 섹션 보고 작업해줘
```

---

### 상황 2: 파일은 많은데 문서가 없을 때

**최선**: 폴더 경로 + 설명
```
packages/@giwoong-ryu-n8n-skillset/ 폴더에서
작업하던 거 이어가줘.
다음엔 Package 2 만들어야 해.
```

---

### 상황 3: 복잡한 프로젝트

**최선**: 세션 백업 활용
```
claude-sessions-backup/ 폴더에서
가장 최근 세션 파일 읽고
마지막 작업 이어가줘
```

---

### 상황 4: 작업 중단점을 기억할 때

**최선**: 구체적 작업 명시
```
@giwoong-ryu/n8n-skillset 패키지는 완성했고,
이제 Package 2 (@giwoong-ryu/korean-content-creator)
만들어야 해. 시작해줘.
```

## 🛠️ 세션 백업 활용법

### 백업 파일 찾기

```bash
# 세션 백업 폴더 확인
ls -lh claude-sessions-backup/

# 가장 최근 파일 찾기
ls -lt claude-sessions-backup/ | head -5
```

### 백업 파일에서 작업 찾기

```bash
# "다음" 또는 "TODO" 검색
grep -n "다음\|TODO\|Next" claude-sessions-backup/*.md

# "완료" 이후 내용 확인
grep -A 5 "완료" claude-sessions-backup/*.md
```

### Claude에게 요청

```
claude-sessions-backup/ 폴더에서
"다음 단계"가 포함된 내용 찾아서
작업 이어가줘
```

## 💡 Pro Tips

### Tip 1: 문서에 "다음 단계" 섹션 남기기

작업 종료 전 항상:
```markdown
## 🚀 다음 단계

1. [ ] Package 2 생성
2. [ ] Windows 스킬 분석
3. [ ] NPM 배포
```

### Tip 2: TODO 주석 활용

코드에 TODO 남기기:
```javascript
// TODO: Package 2 (korean-content-creator) 생성
// TODO: viral-marketing-mastery 스킬 추가
```

### Tip 3: 커밋 메시지에 다음 작업 명시

```bash
git commit -m "Add: Package 1 완성

다음 작업: Package 2 (korean-content-creator) 생성"
```

### Tip 4: 세션 종료 전 요약 요청

```
지금까지 한 작업 요약하고
다음 세션에서 할 일 정리해줘
```

## 🎬 실전 예시

### 예시 1: 개발 프로젝트

**이전 세션**:
- React 컴포넌트 5개 완성
- API 연동 2개 남음

**새 세션 요청**:
```
React 프로젝트 작업 이어가려고 해.
components/ 폴더에 5개 컴포넌트는 완성했고,
이제 API 연동 2개 (auth.ts, users.ts) 만들어야 해.
services/ 폴더에서 시작해줘.
```

- README.md 80% 완성
- 설치 가이드 섹션 남음

**새 세션 요청**:
```
README.md 작업 이어가줘.
현재 "Features" 섹션까지 완성했고,
"Installation" 섹션부터 작성해야 해.
```

**새 세션 요청**:
```
docs/plugin-design-proposal.md 읽고
"다음 단계" 섹션의 2번
"Windows 스킬 분석" 작업 시작해줘.
```

## 📚 관련 도구

### 1. 세션 백업 도구

```bash
# 설치
pipx install claude-conversation-extractor

# 세션 추출
claude-extract --recent 5 --output claude-sessions-backup/

# 검색
claude-search "다음 단계"
```

### 2. Git 활용

```bash
# 마지막 커밋 메시지 확인
git log -1

# 최근 변경 파일 확인
git diff HEAD~1 --name-only

# 브랜치 목록
git branch -a
```

### 3. 파일 검색

```bash
# "다음" 키워드 검색
grep -r "다음\|Next\|TODO" --include="*.md"

# 최근 수정 파일
find . -type f -mtime -1
```
