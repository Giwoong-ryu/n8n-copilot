🎯 Claude 도구 이해하기

"내가 얼마나 신경써야 하는가?" - 쉬운 것부터 단계별로

📚 도구 종류와 설명

각 도구의 역할과 사용 시나리오를 이해하고 필요한 것을 선택하세요

💬
기본 대화
설정 없음 - 질문하면 답변
예시: "이 함수 설명해줘", "Python 문법 알려줘"
📋
LEVEL 0
Projects
간단한 질문이 아닌, 여러 대화에 걸친 큰 작업에서 맥락과 규칙을 계속 기억하게 만드는 기능
설명: 프로젝트를 만들어 Custom Instructions에 작업 방식을 저장하면, 매번 같은 설명을 반복하지 않아도 Claude가 자동으로 규칙을 적용합니다.
구체적 예시:
"월간보고서" 프로젝트 생성 → Custom Instructions에 "주요성과-이슈-계획 순서로 작성, 간결한 톤, 5개 이하 bullet points" 규칙 저장
→ 이후 "3월 보고서 작성해줘" 입력만 하면 저장된 형식으로 자동 작성
💡 비유: 단골 가게 - 내 취향을 이미 알고 있어서 주문이 간편함
⌨️
LEVEL 1
Commands (커맨드)
자주 하는 반복 작업을 한 줄 명령어로 만들어서 즉시 실행하는 기능
설명: 슬래시 명령어(/)를 만들어 복잡한 작업을 한 번에 실행합니다. 매번 긴 지시문을 타이핑하지 않고 짧은 명령어만 입력하면 됩니다.
구체적 예시:
/document 커맨드 생성 → "프로젝트 README 작성 + 설치 방법 추가 + 사용 예시 포함" 작업 정의
→ 채팅창에 /document 한 번 입력하면 자동으로 전체 README.md 파일 생성
💡 비유: 리모콘 버튼 - 내가 눌러야 작동하지만, 한 번에 여러 동작 실행
🤖
LEVEL 2
Agents (에이전트)
복잡한 다단계 작업을 목표만 말하면 스스로 판단하며 자동으로 끝까지 실행하는 기능
설명: 단계별 지시 없이 최종 목표만 알려주면, Agent가 필요한 작업들을 스스로 파악하고 순서대로 실행합니다.
구체적 예시:
"이 문서 검토해줘"라고 요청 → document-reviewer 에이전트가 자동으로:
① 문서 전체 분석 → ② 오류/개선점 탐지 → ③ 수정 제안 작성 → ④ 검토 보고서 생성까지 완료
💡 비유: 로봇 청소기 - 시작만 시키면 알아서 방 구석구석 청소
📚
LEVEL 3
Skills (스킬)
특정 분야의 전문 지식을 Claude에 추가해서, 관련 질문에 더 정확하고 상세하게 답변하게 만드는 기능
설명: Skill을 설치하면 Claude가 대화 중 필요할 때 자동으로 해당 분야 지식을 활용합니다. (Progressive Disclosure로 필요한 부분만 로드)
구체적 예시:
n8n Skill 설치 → "webhook 트리거로 Slack 메시지 보내는 워크플로우 만들어줘" 요청
→ n8n의 HTTP Request 노드, Webhook 트리거, Slack 노드 등 특화 지식을 자동 적용하여 정확한 노드 구성 제공
💡 비유: 전문 자격증 - 가지고만 있어도 해당 분야 실력이 향상됨
📦
올인원 패키지
Plugins (플러그인)
Commands + Agents + Skills를 특정 작업에 맞게 패키징한 종합 도구 세트
설명: 하나의 플러그인을 설치하면 관련된 여러 Commands, Agents, Skills가 한 번에 설치되어 특정 분야 작업을 통합적으로 지원합니다.
구체적 예시:
business-suite 플러그인 설치 시 포함되는 것:
• Business Writing Skill (보고서 작성 지식)
• /weekly-report Command (주간 보고서 생성)
• Document-reviewer Agent (문서 검토)
→ 업무 문서 작성에 필요한 모든 도구를 한 번에 사용 가능
💡 비유: 툴박스 세트 - 드라이버, 망치, 렌치를 따로 사지 않고 세트로 구매
🔌
별도 레이어
MCP (Model Context Protocol)
Claude가 파일, 데이터베이스, API 등 외부 데이터를 실시간으로 직접 읽고 쓸 수 있게 연결하는 기능
설명: MCP 서버를 연결하면 Claude가 복사-붙여넣기 없이 파일, Git 저장소, 데이터베이스를 직접 읽고 수정할 수 있습니다.
구체적 예시:
파일시스템 MCP 연결 → "이 프로젝트의 .env.example 파일 내용 보여줘" 요청
→ Claude가 직접 파일 시스템에 접근해서 실시간으로 파일을 읽고 최신 내용 표시
(파일 내용을 일일이 복사-붙여넣기할 필요 없음)
💡 비유: 인터넷 연결 - 연결되어 있어야 항상 최신 정보를 볼 수 있음

🔄 실전 사용 예시 (복사해서 사용하세요)

📝 보고서 자동 생성

상황: 매주 월요일 주간 보고서를 작성해야 함

방법 1 - Command 사용:
채팅창에 /document 주간 보고서 작성해줘 입력
방법 2 - 자연어로 요청:
"커맨드를 사용해서 이번 주 보고서 자동으로 만들어줘" 입력

🔧 작업 내용 자동 저장

상황: 하루 작업 내용을 정리해서 저장하는 게 번거로움

방법 1 - Command로 간단히:
/save-work 입력 → Claude가 작업 내용 분석하여 정리 및 저장
방법 2 - 상세 지시:
"오늘 작업한 내용 정리해서 저장해줘. 형식은 날짜-제목-내용 순서로" 입력

🎨 발표 자료 템플릿 생성

상황: 매주 사용할 회의 발표 자료 템플릿이 필요함

사전 준비: Presentation Skill 설치 (한 번만)
실행:
"주간 회의용 발표 자료 템플릿 만들어줘. 표지, 주요 성과, 이슈, 계획 슬라이드 포함" 입력
→ Presentation Skill이 자동 활용되어 정확한 템플릿 생성

⚠️ 자주 하는 실수 & 해결법

❌ 실수 1: 긴 대화 세션 계속 사용

문제: 컨텍스트가 계속 누적 → 토큰 소비 증가 → 응답 느려짐
✅ 해결: 큰 작업 끝나면 새 채팅 시작 or /index로 컨텍스트 리셋

❌ 실수 2: MCP 없이 반복 데이터 조회

문제: 매일 같은 파일 내용 복사 붙여넣기 → 매일 30K 토큰 낭비
✅ 해결: MCP 파일시스템 서버 설정 → 필요할 때만 자동 조회 (5K 토큰/일)

❌ 실수 3: 같은 질문 반복

문제: "주간 보고서 작성해줘"를 하루에 20번 → 40K 토큰 낭비
✅ 해결: /weekly-report Commands 만들기 → 10K 토큰으로 감소 (75% 절약)

⌨️ 슬래시 커맨드 (17개)

커맨드 용도 상세 설명
/analyze 코드/시스템 분석 코드베이스 전체를 분석하여 아키텍처 패턴, 의존성, 잠재적 문제점 탐지. 복잡도 분석 및 리팩토링 제안 제공
/build 빌드/컴파일 실행 프로젝트 빌드 자동화. 빌드 에러 발견 시 자동 진단 및 수정 제안. npm build, cargo build 등 자동 감지
/cleanup 코드 정리/포맷팅 불필요한 코드 제거, 포맷 자동 정리, import 최적화, 주석 정리. Prettier/ESLint 자동 실행
/design 아키텍처/시스템 설계 새 기능 또는 시스템 설계. 아키텍처 다이어그램 생성, 컴포넌트 구조 제안, DB 스키마 설계
/document 문서 자동 생성 README, API 문서, JSDoc/Docstring 자동 생성. 코드 분석 후 사용법, 예제 코드 포함
/estimate 작업 추정 및 계획 기능 개발 시간 추정, 작업 분해, 우선순위 제안. 스토리 포인트 및 리스크 분석 포함
/explain 코드/개념 설명 복잡한 코드 로직을 쉬운 언어로 설명. 알고리즘, 디자인 패턴, 라이브러리 사용법 상세 해설
/save-work 작업 내용 자동 저장 하루 작업 내용 자동 정리 및 요약, 주요 변경사항 기록, 날짜별 히스토리 관리
/implement 기능 구현 요구사항 기반 기능 코드 작성. 베스트 프랙티스 적용, 타입 안전성 보장, 에러 핸들링 포함
/improve 코드 개선 제안 성능 최적화, 가독성 향상, 보안 강화 제안. SOLID 원칙 적용 및 코드 스멜 제거
/index 프로젝트 인덱싱 코드베이스 전체 구조 파악, 파일 관계 맵핑, 주요 엔트리 포인트 식별. 빠른 컨텍스트 로딩
/load 컨텍스트 로드 특정 파일/폴더를 작업 컨텍스트에 로드. 관련 파일 자동 탐지 및 함께 로드
/spawn 작업 생성/관리 새 파일/컴포넌트 스캐폴딩, 보일러플레이트 자동 생성. 템플릿 기반 코드 생성
/task 작업 실행 복잡한 멀티스텝 작업 자동 수행. Agent 시스템과 연동하여 자율적 작업 처리
/test 테스트 생성/실행 유닛/통합 테스트 자동 생성. Jest, Pytest 등 자동 감지. 테스트 커버리지 향상 제안
/troubleshoot 문제 진단 및 해결 버그 자동 진단, 로그 분석, 에러 트레이스 추적. 수정 방법 제안 및 즉시 적용 가능
/workflow 워크플로우 자동화 CI/CD 파이프라인 생성, GitHub Actions 설정, 배포 자동화. 개발 워크플로우 전체 자동화

💡 사용법: 슬래시 커맨드는 채팅창에 /커맨드명 형태로 입력하여 특정 작업을 빠르게 실행할 수 있습니다.
