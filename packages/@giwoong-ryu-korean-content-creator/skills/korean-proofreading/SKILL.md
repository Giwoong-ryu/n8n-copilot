# Korean Proofreading (한국어 교정 및 검수)

## Purpose
Provide comprehensive Korean language proofreading expertise covering grammar, spelling, sentence structure, readability, and stylistic refinement for professional content creation.

## When to Use
- Proofreading blog posts, articles, and essays
- Editing marketing copy and social media content
- Reviewing formal documents and business writing
- Improving readability and flow of Korean text
- Correcting common Korean grammar mistakes
- Refining tone and style consistency

## Core Concepts

### 1. Korean Grammar Fundamentals

**조사 (Particles) - Most Common Errors**:

```markdown
주격 조사 (Subject Markers):
✅ 철수가 학교에 갔다 (animate subject)
✅ 비가 온다 (inanimate subject)
❌ 철수이 학교에 갔다 (wrong)

목적격 조사 (Object Markers):
✅ 밥을 먹다 (받침 O)
✅ 물을 마시다 (받침 O)
✅ 커피를 마시다 (받침 X)
❌ 밥를 먹다 (wrong)

처소격 조사 (Location Markers):
✅ 학교에 가다 (direction)
✅ 학교에서 공부하다 (location of action)
❌ 학교를 가다 (incorrect particle)
```

**어미 (Endings) - Formal vs Informal**:

| 상황 | 격식체 | 비격식체 | 반말 |
|------|--------|----------|------|
| 평서문 | 합니다/입니다 | 해요/이에요 | 해/이야 |
| 의문문 | 합니까?/입니까? | 해요?/이에요? | 해?/이야? |
| 명령문 | 하십시오 | 하세요 | 해 |
| 청유문 | 합시다 | 해요 | 하자 |

### 2. 맞춤법 (Spelling) Common Errors

**띄어쓰기 (Spacing)**:

```markdown
의존명사 (Bound Nouns) - 띄어쓰기 필수:
✅ 할 수 있다 (O)
❌ 할수있다 (X)

✅ 먹을 것이다 (O)
❌ 먹을것이다 (X)

✅ 5개월 만에 (O)
❌ 5개월만에 (X)

단위 명사 - 붙여쓰기:
✅ 사과 한개 (X) → 사과 한 개 (O)
✅ 10명 (O)
✅ 3시간 (O)

보조용언 - 띄어쓰기 원칙적, 붙여쓰기 허용:
✅ 먹어 버렸다 (권장)
✅ 먹어버렸다 (허용)
```

**헷갈리는 맞춤법**:

| 틀린 표현 | 올바른 표현 | 설명 |
|-----------|-------------|------|
| 되 | 돼 | "되어"의 줄임말 |
| 왠지 | 웬지 | "무슨 까닭인지" |
| 왠만하면 | 웬만하면 | "어지간하면" |
| 않됐다 | 안 됐다 | 부정 부사는 띄어쓰기 |
| 금방 | 금방 | "곧, 조금 전" (둘 다 맞음) |
| 어떻해 | 어떡해 | "어떻게 해"의 줄임말 |
| 가르키다 | 가리키다 | "지시하다" |
| 틀리다 | 다르다 | "틀림" ≠ "다름" |

**맞춤법 규칙**:

```markdown
Rule 1: 사이시옷
✅ 나뭇가지, 핏줄, 곳간, 숫자
❌ 나무가지, 피줄, 곧간, 수자

Rule 2: 두음법칙 (Initial Sound Rule)
✅ 여자 (女子), 이론 (理論), 연습 (練習)
❌ 녀자, 니론, 년습

단, 복합어에서는 원음 표기:
✅ 남여 (남녀), 이니론 (이理論)

Rule 3: 사동/피동 접미사
-이- : 먹이다, 읽히다
-히- : 입히다, 잡히다
-리- : 밀리다, 놀리다
-기- : 안기다, 덮이다
-우- : 맞추다, 돋우다
-구- : 돋구다
-추- : 맞추다
```

### 3. 문장 구조 (Sentence Structure)

**Readability Principles**:

**1. 문장 길이 최적화**:
```markdown
짧은 문장 (15자 이하):
- 사용처: SNS, 광고 카피, 중요 포인트
- 예: "지금 시작하세요. 무료입니다."

적정 문장 (20-40자):
- 사용처: 일반 콘텐츠, 블로그 본문
- 예: "오늘 소개할 제품은 사용이 간편하고 가격도 합리적입니다."

긴 문장 (40자 이상):
- 사용처: 학술, 공식 문서 (피할 수 있으면 피하기)
- 예: "본 연구에서는 A 이론을 바탕으로 B 방법론을 활용하여 C 결과를 도출하였으며, 이는 D 선행 연구와 일치한다."
  → 2-3문장으로 나누기 권장
```

**2. 주어-서술어 호응**:
```markdown
❌ 나의 목표는 건강해지는 것입니다. (X)
    (목표 ≠ 건강해지다)
✅ 나의 목표는 건강해지는 것이다. (O)
✅ 나는 건강해지는 것을 목표로 한다. (O)

❌ 이 책의 장점은 쉽게 읽힙니다. (X)
    (장점 ≠ 읽히다)
✅ 이 책의 장점은 쉽게 읽힌다는 점이다. (O)
✅ 이 책은 쉽게 읽힌다는 장점이 있다. (O)
```

**3. 문장 성분 순서**:
```markdown
기본 어순: 주어 - 부사어 - 목적어 - 서술어

자연스러운 순서:
✅ 철수가 어제 학교에서 친구를 만났다.
   (주어) (시간) (장소) (목적어) (서술어)

강조를 위한 도치:
✅ 친구를 만났다, 철수가 어제 학교에서.
   (목적어 강조)
```

### 4. 가독성 (Readability)

**단락 구성**:

```markdown
단락 구조:
┌─────────────────────────┐
│ [주제문] Topic Sentence │
│  - 단락의 핵심 내용      │
├─────────────────────────┤
│ [뒷받침 문장] Support    │
│  - 예시, 설명, 근거      │
│  - 3-5 문장              │
├─────────────────────────┤
│ [마무리 문장] Closing    │
│  - 요약 또는 전환        │
└─────────────────────────┘

단락 길이:
- 단문 단락: 1-2문장 (강조, 전환)
- 표준 단락: 3-5문장 (일반적)
- 장문 단락: 6-8문장 (상세 설명)

⚠️ 8문장 초과 시 분리 고려
```

**문장 연결**:

```markdown
접속사 사용:

나열: 그리고, 또한, 더불어, 아울러
대조: 그러나, 하지만, 반면에, 그럼에도
인과: 그래서, 따라서, 그러므로, 그 결과
전환: 한편, 그런데, 그렇지만
예시: 예를 들어, 이를테면, 가령
부연: 즉, 다시 말해, 요컨대

과도한 사용 피하기:
❌ 그래서 나는 학교에 갔다. 그래서 친구를 만났다. 그래서 점심을 먹었다.
✅ 나는 학교에 가서 친구를 만났고, 함께 점심을 먹었다.
```

**반복 회피**:

```markdown
동일 단어 반복:
❌ 이 제품은 좋은 제품입니다. 제품의 품질이 좋습니다.
✅ 이 제품은 품질이 우수합니다. 사용자 만족도도 높습니다.

유사 표현 활용:
- 제품 → 이 아이템, 해당 상품, 이것
- 좋다 → 우수하다, 훌륭하다, 뛰어나다, 만족스럽다
- 많다 → 다양하다, 풍부하다, 다수이다

대명사 활용:
✅ 철수는 공부를 열심히 한다. 그는 매일 도서관에 간다.
   (철수 → 그)
```

### 5. 문체 (Style)

**격식의 정도**:

```markdown
Level 1: 최고 격식 (법률, 공문서)
- 종결어미: ~이다/~임
- 예: "당 법인은 다음 사항을 준수함."

Level 2: 높은 격식 (비즈니스, 학술)
- 종결어미: ~합니다/~입니다
- 예: "본 연구는 다음과 같은 의의가 있습니다."

Level 3: 중간 격식 (블로그, 뉴스)
- 종결어미: ~해요/~이에요
- 예: "오늘은 날씨가 참 좋아요."

Level 4: 낮은 격식 (SNS, 친구)
- 종결어미: ~해/~이야
- 예: "오늘 날씨 진짜 좋아!"

일관성 유지: 한 콘텐츠 내에서 하나의 격식 선택
```

**어감 선택**:

| 중립 | 긍정 | 부정 | 비격식 |
|------|------|------|--------|
| 생각하다 | 고민하다, 숙고하다 | 걱정하다 | 생각해보다 |
| 말하다 | 제안하다, 설명하다 | 따지다, 주장하다 | 얘기하다 |
| 보다 | 감상하다, 관람하다 | 훔쳐보다 | 구경하다 |
| 먹다 | 맛보다, 즐기다 | 처먹다 | 먹어버리다 |

## Examples

### Example 1: 블로그 포스팅 교정

**Before (교정 전)**:
```
제목: 제주도여행후기

어제 제주도에 다녀왔다. 날씨가 좋았다. 그래서 기분이 좋았다.
성산일출봉에 갔다. 경치가 아름다웠다. 사진을 많이 찍었다.
다음에 또 가고싶다.

(문제점)
- 제목 띄어쓰기 오류
- 문장이 너무 짧고 단조로움
- 접속사 남용
- 구체성 부족
- 맞춤법 오류 ("가고싶다")
```

**After (교정 후)**:
```
제목: 제주도 여행 후기 - 성산일출봉에서의 완벽한 하루

어제 제주도를 다녀왔는데, 날씨가 정말 완벽했어요.
맑은 하늘 아래서 성산일출봉에 올랐는데, 정상에서 본 경치가
정말 아름다웠습니다. 푸른 바다와 제주의 오름들이 한눈에 들어와
계속 감탄이 나왔어요.

특히 일출 시간(오전 6시 30분)에 맞춰 가니 더욱 특별했습니다.
등반은 왕복 40분 정도 소요됐는데, 중간중간 쉬어가며
사진도 많이 찍었어요.

다음에는 섭지코지와 우도도 함께 둘러보고 싶네요!

(개선사항)
✅ 제목 띄어쓰기 수정 + 부제목 추가
✅ 문장 결합으로 자연스러운 흐름
✅ 구체적 정보 추가 (시간, 소요시간)
✅ 감정 표현 강화 ("정말", "완벽했어요")
✅ 맞춤법 수정 ("가고 싶다")
✅ 격식 통일 (해요체)
```

## Common Patterns

### Pattern 1: 교정 순서

```markdown
1단계: 맞춤법 검사 (5분)
- 네이버 맞춤법 검사기 활용
- 부산대 한국어 맞춤법 검사기
- 기본적인 오류 수정

2단계: 문장 구조 검토 (10분)
- 주어-서술어 호응 확인
- 문장 길이 조정 (40자 이하 권장)
- 단락 나누기

3단계: 가독성 개선 (10분)
- 반복 표현 제거
- 접속사 적절성 확인
- 문장 연결 자연스럽게

4단계: 톤 & 스타일 통일 (5분)
- 격식 일관성 (해요체/합니다체)
- 어감 조정 (긍정/부정/중립)
- 이모지 적절성 (SNS만)

5단계: 최종 검토 (5분)
- 소리 내어 읽기 (흐름 확인)
- 핵심 메시지 명확성 확인
- 오탈자 재점검
```

### Pattern 2: 가독성 향상 기법

```markdown
Technique 1: 문장 분리
Before: "이 제품은 사용이 간편하고 가격도 저렴하며
         디자인도 예쁘고 기능도 다양해서 추천합니다."
After:  "이 제품은 사용이 간편합니다.
         가격도 저렴하고, 디자인도 예쁩니다.
         다양한 기능까지 갖춰서 정말 추천해요!"

Technique 2: 능동태 사용
Before: "이 문제는 빠르게 해결될 것으로 기대됩니다."
After:  "이 문제는 빠르게 해결될 것입니다."
        or "저희가 이 문제를 빠르게 해결하겠습니다."

Technique 3: 구체적 표현
Before: "가격이 저렴합니다."
After:  "정가 5만원에서 3만5천원으로 30% 할인되었습니다."

Technique 4: 불필요한 수식어 제거
Before: "정말 매우 아주 굉장히 좋습니다."
After:  "정말 좋습니다." or "탁월합니다."
```

## Best Practices

### ✅ DO

**교정 전 준비**:
- **목적 파악**: 블로그? SNS? 공식 문서?
- **타겟 이해**: 20대? 40대? 전문가?
- **톤 확인**: 격식? 친근? 전문적?
- **플랫폼 고려**: 모바일? 데스크톱?

**교정 중**:
- **맞춤법 우선**: 기본적인 오류 먼저 수정
- **소리 내어 읽기**: 자연스러운 흐름 확인
- **객관적 시선**: 독자 입장에서 읽기
- **일관성 유지**: 격식, 어감, 스타일 통일
- **구체성 추가**: 숫자, 예시, 디테일

**교정 후**:
- **재검토**: 최소 1시간 후 다시 읽기
- **타인 피드백**: 가능하면 2명 이상 리뷰
- **버전 관리**: 수정 전/후 버전 보관
- **학습**: 자주 하는 실수 노트 정리

### ❌ DON'T

**절대 금지**:
- **무분별한 수정**: 원문 의도 왜곡하지 않기
- **과도한 수식**: "매우 아주 정말" 과다 사용
- **격식 혼재**: 해요체 + 합니다체 섞지 않기
- **맥락 무시**: 앞뒤 문맥 고려 없는 수정
- **AI 의존**: 맞춤법 검사기 맹신하지 않기

**피해야 할 실수**:
- 띄어쓰기만 수정하고 끝내기
- 문장 구조 개선 없이 단어만 바꾸기
- 독자 고려 없이 자기 스타일 강요
- 원문 삭제 (항상 원본 보존)
- 시간 압박에 대충 넘어가기

## Troubleshooting

### Issue: "맞춤법 검사기가 틀렸다고 하는데 맞는 것 같아요"

**Cases**:
1. **방언/은어**: 검사기는 표준어만 인식
   - 해결: 콘텐츠 성격에 따라 판단 (SNS는 허용 가능)

2. **신조어**: 검사기 업데이트 지연
   - 예: "갓생", "빡공", "꾸안꾸"
   - 해결: 타겟 이해 시 사용 OK, 설명 추가 권장

3. **구어체 표현**: 입말을 그대로 쓴 경우
   - 예: "그쵸", "그럼요", "맞쥬"
   - 해결: SNS/블로그는 허용, 공식 문서는 수정

4. **의도적 표현**: 강조를 위한 표기
   - 예: "완전 대애박", "진짜아~~"
   - 해결: 브랜드 톤에 맞으면 유지

**Rule**: 검사기는 참고용, 최종 판단은 사람이

### Issue: "문장을 어떻게 고쳐야 할지 모르겠어요"

**Diagnosis Process**:
```
문장: "이 제품이 좋습니다."

Step 1: 무엇이 문제인가?
→ 너무 평범함, 구체성 부족

Step 2: 핵심 메시지는?
→ 제품의 '어떤 점'이 좋은가?

Step 3: 디테일 추가
→ "이 제품은 사용이 간편해서 좋습니다."

Step 4: 감정 추가
→ "이 제품은 사용이 정말 간편해서 만족스럽습니다."

Step 5: 증거 추가
→ "이 제품은 버튼 하나로 작동되어
   사용이 정말 간편합니다.
   처음 써보는 날부터 바로 적응할 수 있었어요."
```

### Issue: "같은 단어가 계속 반복되는데 대체어를 모르겠어요"

**Solution: 유의어 사전 활용**

```markdown
좋다 → 훌륭하다, 우수하다, 뛰어나다, 탁월하다,
       만족스럽다, 괜찮다, 나쁘지 않다

나쁘다 → 좋지 않다, 미흡하다, 부족하다, 아쉽다,
        개선이 필요하다, 기대에 못 미치다

크다 → 거대하다, 방대하다, 규모가 있다, 넓다

작다 → 소형이다, 작은 편이다, 아담하다, 간소하다

빠르다 → 신속하다, 즉각적이다, 재빠르다, 날쌘

느리다 → 더디다, 지체되다, 천천히 진행되다
```

**Context-based 선택**:
```
긍정 강조: 훌륭하다, 탁월하다
중립: 괜찮다, 나쁘지 않다
조심스러운 긍정: 만족스럽다, 좋은 편이다
```
